SpringData疑问合集

1、为什么在springdata jpa接口不用@Repository还能Autowired成功

a.首先，把问题用百度搜索，发现没有任何卵用，全是互相抄的乱七八糟的博客
b.然后，科学上网，在谷歌搜索”springdata don’t have repository”，第一条就指向stackoverflow，

c. 关于”it’s created to add functionality for derived class.”的理解没有特别明确，于是百度”spring Data @NoRepositoryBean”，找到了相关解释:确保添加了该注解的 repository 接口不会在运行时被创建实例。也就是说，使用了该注解的接口不会被单独创建实例，只会作为其他接口的父接口而被使用。
和springdata的重要接口Repository(https://blog.csdn.net/crpxnmmafq/article/details/78147251)
d. 个人总结一下：
@NoRepositoryBean 是不允许spring容器直接创建该接口的实例
@Repository，SpringData的基础接口，凡是继承这个接口的，框架会帮我们去创建相关的实例

2、在JPQL中直接使用”?”占位符报错：JDBC style parameters (?) are not supported for JPA queries.

解决： 疑似新版本jar包不支持(参考博客)这种语法，改成”:paramName”的方式，就可以正常调用。
